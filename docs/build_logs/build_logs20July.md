**Duration:** 2 Hours

### Goal

### Problem Encountered

- Correct page insertion and update in `PageStorage`.
- Accurate page count maintenance.
- FIFO behavior of the `Frontier`.

- Successful HTML retrieval using `Fetcher`.

### Outcome
Confirmed that all implemented components function correctly when tested independently and are ready for integration into the crawler.

### Key Learning

## Session 2

**Duration:** 2 Hours

### Goal
Validate the correctness of the URL normalization component by testing it against various URL formats encountered during crawling.

### Problem Encountered
Several URL formats required careful handling, including relative paths, duplicate slashes, fragments, protocol-relative URLs, and unsupported protocols. Ensuring that all edge cases produced consistent normalized URLs required extensive testing.

### What I Tried
- Prepared multiple test cases covering different URL formats.
- Tested normalization of absolute and relative URLs.
- Verified handling of `.` and `..` path segments.
- Checked removal of fragments.
- Tested rejection of unsupported protocols such as `javascript:`, `mailto:`, and `tel:`.
- Evaluated protocol-relative URLs and query-only URLs.

### Solution
Identified and corrected issues in the normalization logic, including handling of duplicate slashes and uppercase URL schemes. Re-ran the complete test suite to verify the fixes.

### Outcome
Successfully validated the `URLNormalizer` implementation against a wide range of input URLs, confirming that the component behaves correctly for the crawler's requirements.

### Key Learning
URL normalization involves numerous edge cases, making comprehensive testing essential to ensure consistent crawling behavior.
